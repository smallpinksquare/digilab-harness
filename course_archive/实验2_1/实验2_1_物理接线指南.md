# 实验 2_1 物理接线指南（按芯片逐 pin 对照）

> 仿真已 16/16 PASS（见 `verify_report.json`）。本电路只用 1 颗 74151，**不需要任何 7400/7420**。如果照 `circuit.txt` 接但不通，多半是踩了下面"易错点 ★"里的坑。

## 0. 信号速查（电路里一共 6 条信号）

| 信号 | 含义 | 由谁产生 |
|------|------|----------|
| `A1` | 实验输入：输血者血型高位 | 拨码开关 / 信号源 |
| `A0` | 实验输入：输血者血型低位 | 拨码开关 / 信号源 |
| `B1` | 实验输入：受血者血型高位 | 拨码开关 / 信号源 |
| `B0` | 实验输入：受血者血型低位 | 拨码开关 / 信号源 |
| `F`  | 输出：1 = 配型成功 | 74151 pin 5 |
| `VCC/GND` | 电源 +5V / 地 | 电源 |

> 血型编码：`00=O, 01=A, 10=B, 11=AB`。规则等价于"输血者抗原集合 ⊆ 受血者抗原集合"。

| pin | 引脚角色 | 应接 | 物理信号 / 来源 | 说明 |
|:---:|---------|------|-----------------|------|
| 1   | D3（数据）| INPUT **B0** | `B0`，菊花链起点 | ★ 见易错点 1（D3=B0）|
| 2   | D2（数据）| **本片 pin 1** | `B0`，菊花链 | D2=B0，跳线接到 pin 1 |

## 2. 易错点 ★（按出错频率排序）

### ★ 易错点 1：4 个变量到底哪 3 个当 select、哪 1 个当数据

**本电路的约定**（必须严格按这个，否则真值表全错）：

- select：`S2 S1 S0 = A1, A0, B1` → 对应 74151 的 `pin 9, pin 10, pin 11`
- 数据：8 个 `Di` 按公式取值，其中 D2/D3/D7 取 `B0`，**B0 不是 select**

如果你不小心把 B0 当 select 了（比如接到 pin 11），整张表会按 `S2S1S0=A1 A0 B0` 重新解释，所有 16 行都会错。

### ★ 易错点 2：74151 的 select 顺序 `C, B, A` = `MSB, mid, LSB`

74151 datasheet 把 select 端标为 `C, B, A`，**`C` 是 MSB**：

- pin 9 = C = S2（最高位）
- pin 10 = B = S1
- pin 11 = A = S0（最低位）

如果你照"字面 ABC"反着接（A1→pin 11, A0→pin 10, B1→pin 9），select 索引会按二进制位反过来理解，D0↔D4、D1↔D5、D2↔D6、D3↔D7 会全交换，结果不对。

> **正确接法**：实验 `A1 → pin 9`，`A0 → pin 10`，`B1 → pin 11`。

### ★ 易错点 3：菊花链 = "短接同一信号"，不是"信号经过加工"

`circuit.txt` 里这几行：

```
74151_A.1 -> 74151_A.2
74151_A.2 -> 74151_A.12
74151_A.3 -> 74151_A.4
74151_A.4 -> 74151_A.14
74151_A.13 -> 74151_A.15
```

**不是**意味着 pin 1 的输出送到 pin 2（D 系列都是输入端，没有输出！），而是：

> "这几个 pin 在物理上接在同一根线上，承载同一个恒定信号"

物理实操：先把 `INPUT.B0` 拉一根线到 `pin 1`，再用短跳线把 `pin 1 → pin 2`、`pin 2 → pin 12` 串起来——三个 pin 自然就在同一信号 B0 上。VCC 和 GND 那两条菊花链同理。

| 信号 | 物理走线（沿这条线串起来即可） | 涉及数据端 |
|------|--------------------------------|------------|
| `B0` | `INPUT.B0 → pin 1 → pin 2 → pin 12` | D3, D2, D7 |
| `1`  | `VCC.0 → pin 3 → pin 4 → pin 14`    | D1, D0, D5 |
| `0`  | `GND.0 → pin 13 → pin 15`           | D6, D4 |

### ★ 易错点 4：使能引脚漏接

74151 的 `Ē`（pin 7）必须接 GND，否则**无论 select 取何值，Y 永远输出 0、Ȳ 永远输出 1**。

### ★ 易错点 5：电源/地全接

- pin 16 → VCC
- pin 8 → GND
- pin 7 (Ē) → GND（这条单独再强调一次，最容易忘）

## 3. 上电前最终自检表（5 分钟）

按下面顺序逐项打勾：

- [ ] 芯片缺口/小圆点朝上（按 datasheet 标准方向）
- [ ] pin 16 → VCC，pin 8 → GND，pin 7 → GND（**3 条电源/使能线，缺一不可**）
- [ ] **select 接法**：`A1 → pin 9`、`A0 → pin 10`、`B1 → pin 11`（不是反着！）
- [ ] **B0 不是 select**：`B0 → pin 1`，再用跳线串到 `pin 2`、`pin 12`（共 3 处）
- [ ] **VCC 菊花链**：`VCC → pin 3 → pin 4 → pin 14`（D1, D0, D5 都在 VCC 上）
- [ ] **GND 菊花链**：`GND → pin 13 → pin 15`（D6, D4 都在 GND 上）
- [ ] 输出探针：F 接 pin 5（不是 pin 6！pin 6 是反相输出，会显示反的结果）

通过后再上电，按下表逐行触发：

| A1 | A0 | B1 | B0 | 期望 F | 含义 |
|:--:|:--:|:--:|:--:|:------:|------|
| 0 | 0 | 0 | 0 | 1 | O→O 成功 |
| 0 | 0 | 0 | 1 | 1 | O→A 成功 |
| 0 | 0 | 1 | 0 | 1 | O→B 成功 |
| 0 | 0 | 1 | 1 | 1 | O→AB 成功 |
| 0 | 1 | 0 | 0 | 0 | A→O 失败 |
| 0 | 1 | 0 | 1 | 1 | A→A 成功 |
| 0 | 1 | 1 | 0 | 0 | A→B 失败 |
| 0 | 1 | 1 | 1 | 1 | A→AB 成功 |
| 1 | 0 | 0 | 0 | 0 | B→O 失败 |
| 1 | 0 | 0 | 1 | 0 | B→A 失败 |
| 1 | 0 | 1 | 0 | 1 | B→B 成功 |
| 1 | 0 | 1 | 1 | 1 | B→AB 成功 |
| 1 | 1 | 0 | 0 | 0 | AB→O 失败 |
| 1 | 1 | 0 | 1 | 0 | AB→A 失败 |
| 1 | 1 | 1 | 0 | 0 | AB→B 失败 |
| 1 | 1 | 1 | 1 | 1 | AB→AB 成功 |

如果某一行错了，**先观察是不是整列对调（select 接反）**：

- 若 `(A1A0B1)=000~011` 这组（前 8 行）和 `100~111` 这组的输出**整体对调**，是 `A1` 接错位置（接到了 pin 11 而不是 pin 9）。
- 若所有 F 都恒为 0，是 `pin 7 (Ē)` 没接 GND（使能漏接）。
- 若 F 输出与期望全部相反，是把探针接到 pin 6（Ȳ）了，移到 pin 5（Y）即可。

## 4. 实战记录（首次接板踩坑）

> 本次实验首板调试出现的两个真实问题，作为补充警示。

### 现象：F 输出按 `A1A0B1B0=0000..1111` 顺序为 `0010 0011 0010 0011`

**特征**：前 8 行（A1=0 段）和后 8 行（A1=1 段）输出一模一样，A1 翻转完全无效。

**根因 1：A1 那根杜邦线整个忘了接**
- 学生的接法：杜邦线先全部接到副板（开关板）上，然后用排线把副板和主板（74151 那块）连起来。漏插了 A1 这一根的"主板侧"，A1 输入悬空。
- CMOS 输入悬空 → pin 9 一直读到某个稳定值（实测约等于 0），导致只有 D0~D3 在被选中，永远选不到 D4~D7。
- **判别口诀**：**周期 8 行重复 = MSB select（A1）失效**。任何时候在测真值表，看到"前后两半完全相同"就先去检查 MSB select 那根线的两端有没有真插上。

**根因 2：A0 / B1 / B0 select 端的对应顺序接反**
- 学生按"字母顺序"把 A0→pin 11、B1→pin 10、B0→pin 9 这种习惯接，但 74151 的 select 是 `pin 9 = C(MSB), pin 10 = B, pin 11 = A(LSB)`，**和实验里 A1, A0, B1 的语义顺序是反的**。
- 这条本来在 ★ 易错点 2 已经写了。本次再补强一句：**接 select 时按 pin 编号倒着想**——pin 编号从 9 到 11，对应实验变量按"重要性"从高到低（A1, A0, B1）。

### 不用万用表的快速定位办法（已实测有效）

把 F 输出的 LED 那根线临时拔下来，**直接当成"探针"挪到 pin 9**，然后单独拨 A1 开关：

- A1 拨到 1，LED 亮 → A1 这根线通到 pin 9 了
- A1 拨到 1，LED 不亮 → A1 没接上 pin 9（本次就是这个症状）

整个判定 30 秒。同样的探针法可以套到 pin 10 / pin 11 验证 A0、B1 是否到位。

### 修复后

把 A1 那根线补上、A0 / B1 / B0 三根的 pin 顺序按表 1 重新对位插好后，16 行真值表全部一次通过。
