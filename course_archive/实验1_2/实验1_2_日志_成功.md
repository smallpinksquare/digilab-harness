## 任务概述

- **实验**：4 位二进制数值判别电路（判别 $3 \le N \le 6$）
- **源文件**：[实验一/(2)/2.md](../实验一/（2）/2.md)
- **逻辑表达式**（原始 LaTeX）：

$$Y = \overline{\overline{\overline{B_3} \cdot B_2 \cdot \overline{B_1 B_0}} \cdot \overline{\overline{B_3} \cdot \overline{B_2} \cdot B_1 \cdot B_0}}$$

- **输入变量**：`B3, B2, B1, B0`（共 4 个，全部接入芯片）
- **输出变量**：`Y`
- **器件清单**：`7400 × 1, 7420 × 1`

## 算法思路

按附录 A.2 把 LaTeX 表达式逐层翻译成 NAND 原语：

| 子项 | NAND 形式 | 备注 |
|------|-----------|------|
| $\overline{B_3}$ | `NAND2(B3, B3)` | 自反 |
| $\overline{B_2}$ | `NAND2(B2, B2)` | 自反 |
| $\overline{B_1 \cdot B_0}$ | `NAND2(B1, B0)` | |
| $\overline{\overline{B_3} \cdot B_2 \cdot \overline{B_1 B_0}}$ | `NAND4(NAND2(B3,B3), B2, NAND2(B1,B0), 1)` | 3 输入与项 → NAND4 + 拉高 |
| $\overline{\overline{B_3} \cdot \overline{B_2} \cdot B_1 \cdot B_0}$ | `NAND4(NAND2(B3,B3), NAND2(B2,B2), B1, B0)` | 4 输入与项 |
| $Y$（顶层）| `NAND2(...)` | |

完整表达式（见 [真值表与逻辑表达式/实验1_2_表达式.md](../真值表与逻辑表达式/实验1_2_表达式.md)）：

```
Y = NAND2(
      NAND4(NAND2(B3, B3), B2, NAND2(B1, B0), 1),
      NAND4(NAND2(B3, B3), NAND2(B2, B2), B1, B0)
    )
```

### 关键点：自动 CSE

`NAND2(B3, B3)`（即 `~B3`）在表达式里**出现两次**（两个 NAND4 都用到）。如果朴素综合，会消耗 5 个 NAND2 门，超出 1×7400(4 门) 的预算。

synthesizer 的自动 CSE 缓存（`Dict[Node, Endpoint]`）发现两个 `NAND2(B3, B3)` 节点结构相等（frozen dataclass 等价），第二次直接返回第一次分配的输出端点，相当于在电路上做扇出。最终生成的 `~B3` 只占 1 个 NAND2 门，**门输出引脚 `7400_A.3` 通过菊花链先连到 `7420_A.1`，再从那里跳到 `7420_A.9`**（电气上等价于一个门同时驱动两个 NAND4 的输入）。

最终门用量：4 NAND2 + 2 NAND4 = 1×7400(4 门，全用) + 1×7420(2 门，全用)。**门预算用满**。

## 完整电路连接方式（按附录 B.4 排序）

来自 [circuit.txt](circuit.txt)，每行格式 `源.引脚 -> 目的.引脚`。

本电路应用了两项面包板友好的物理优化：

1. **菊花链扇出（daisy-chain fanout）**：同一驱动者的多个消费者依字典序串联，物理上对应"从输入开关拉一根线到第一个芯片，再从那里跳到下一个芯片"的接法。
2. **VCC 输入替换（VCC input substitution）**：原本"NAND4 多余输入接 VCC"的引脚，被短接到该门内首个真实信号输入引脚上。借用 NAND 的代数恒等式 `NAND4(X, Y, Z, X) = NAND3(X, Y, Z)`，物理上避免直接接电源带来的高电平不稳定问题；电气上完全等价。

两项优化对 verifier 都透明（verifier 用并查集合并所有连通端点），verify_report 仍 PASS。

### 菊花链摘要

| 信号 | 物理走线 |
|------|----------|
| `~B3`（CSE 扇出 2 处 + VCC 替换借接 1 处 = 共 3 处） | `7400_A.3 → 7420_A.1 → 7420_A.5 → 7420_A.9` |
| `B3`（自馈到 NAND2 两输入） | `INPUT.B3 → 7400_A.1 → 7400_A.2` |
| `B2`（扇出 3 处） | `INPUT.B2 → 7400_A.9 → 7400_A.10 → 7420_A.2`（其中 `7400_A.9→.10` 是 NAND2(B2,B2) 自馈，`.10→7420_A.2` 是后续菊花段）|
| `B1` / `B0`（各扇出 2 处） | `INPUT.B1 → 7400_A.4 → 7420_A.12` ；`INPUT.B0 → 7400_A.5 → 7420_A.13` |

> 关键体现：`~B3` 这一个门输出现在通过菊花链串联 3 个 7420_A 输入引脚——`7420_A.1` 是第一个 NAND4 的第 1 输入（语义上的 ~B3）、`7420_A.5` 是第一个 NAND4 的第 4 输入（VCC 替换：原本接 VCC 用作 1，现在借接 ~B3 实现等价 NAND3）、`7420_A.9` 是第二个 NAND4 的第 1 输入（CSE 共享）。物理上从 `7400_A.3` 拉 1 根线到 `7420_A.1`，然后用 2 段短跳线 `1→5`、`5→9` 串过去，整组 3 段连线，远比原"星型 + VCC 总线"方案省线。

## 代码变更

本实验本身未触发任何代码改动；依赖以下三批 harness 框架升级，均已合并到原始代码并通过全部单元测试：

1. **首批升级（执行本实验前合入）**：字面量 0/1（VCC/GND）、自动 CSE、中间变量语法、多行表达式解析；其中自动 CSE 是本实验门预算能装下的关键。
2. **菊花链扇出**：在 `synthesizer.py` 末尾加入 `_daisy_chain_connections()` 后处理；dst 为 VCC/GND/NC 的连接保留星型，其余按 src 分组后重组为菊花链。
3. **VCC 输入替换（本次合入）**：在 `synthesizer.py` 的 `_emit_node` 处理 NAND 节点处加入：扫描子端点列表，把所有 `_VCC_EP`（即 `Const(1)` 注入位）替换为该门首个真实信号端点（proxy）。借用 `NAND(X, ..., X) = NAND(X, ...)` 的代数恒等，物理上消除直接接 VCC 带来的高电平不稳定；电气等价。在本实验中，第一个 NAND4 的第 4 输入（原本 = `VCC.0`）被替换为 ~B3（首个子端点 = `7400_A.3`），结合 CSE 共享与菊花链化，~B3 信号沿 `7400_A.3 → 7420_A.1 → 7420_A.5 → 7420_A.9` 一条链覆盖 3 处输入。verifier 仍用并查集合并连通端点，无需修改。
